# Image Hosting

Веб-додаток для завантаження та зберігання зображень з підтримкою пагінації, сортування та видалення.

## Стек технологій

- **Backend:** Python 3.12 (вбудований HTTP-сервер, без фреймворків)
- **База даних:** PostgreSQL
- **Frontend:** HTML / CSS / Vanilla JavaScript
- **Інфраструктура:** Docker, Docker Compose, Nginx (reverse proxy)

## Функціонал

- Завантаження зображень (JPG, PNG, GIF, WEBP) з валідацією типу та розміру
- Перегляд галереї з пагінацією
- Сортування за датою завантаження (нові / старі)
- Перегляд деталей зображення
- Видалення зображень
- Автоматичне резервне копіювання бази даних

## Запуск

### Вимоги

- Docker
- Docker Compose

### Налаштування

1. Скопіюйте файл конфігурації:
   ```bash
   cp .env.example .env
   ```

2. Відредагуйте `.env` — вкажіть параметри бази даних та інші налаштування.

### Запуск проєкту

```bash
make up-build
```

Або без Makefile:

```bash
docker compose up --build
```

Додаток буде доступний за адресою: [http://localhost](http://localhost)

## Структура проєкту

```
image-hosting/
├── backend/        # Python HTTP-сервер, обробники, репозиторій
├── frontend/       # HTML, CSS, JavaScript
├── db/             # SQL-скрипти ініціалізації бази даних
├── backups/        # Скрипти резервного копіювання
├── nginx.conf      # Конфігурація Nginx
└── docker-compose.yml
```

## API

| Метод | Endpoint | Опис |
|-------|----------|------|
| `GET` | `/api/images` | Список зображень (з пагінацією) |
| `GET` | `/api/images/{filename}` | Інформація про зображення |
| `POST` | `/api/upload` | Завантаження нового зображення |
| `DELETE` | `/api/images/{filename}` | Видалення зображення |

### Параметри запиту для `GET /api/images`

| Параметр | Тип | За замовчуванням | Опис |
|----------|-----|-----------------|------|
| `page` | int | `1` | Номер сторінки |
| `limit` | int | `8` | Кількість елементів на сторінці |
| `order` | string | — | Сортування: `asc` або `desc` |
